# Build Binance-Style Token Swap Application

## 1. Project Initialization & Configuration
- **Initialize Vite Project**: Create a new React + TypeScript project in `src/problem2`.
- **Install Dependencies**:
  - `tailwindcss`, `postcss`, `autoprefixer` for styling.
  - `lucide-react` for icons.
  - `clsx`, `tailwind-merge` for class management.
  - `zod` for validation.
  - `framer-motion` (optional, for smooth modal/accordion animations if needed, otherwise CSS transitions).
- **Configure Tailwind**: Update `tailwind.config.js` with the specified Binance color palette (Primary `#FCD535`, Background `#1E2329`, etc.) and fonts.

## 2. Architecture & Folder Structure
Create the following modular structure:
- `src/components/layout`: `Layout.tsx`, `Header.tsx`, `Sidebar.tsx`
- `src/components/swap`: `SwapCard.tsx`, `TokenInput.tsx`, `SlippageSettings.tsx`, `OrderDetails.tsx`
- `src/components/ui`: `Button.tsx`, `Modal.tsx`, `Skeleton.tsx`, `Toast.tsx`
- `src/hooks`: `useSwap.ts` (logic), `useToast.ts` (notifications)
- `src/constants`: `tokens.ts` (mock data), `theme.ts`
- `src/types`: `index.ts` (shared interfaces)
- `src/utils`: `format.ts` (number formatting)

## 3. Implementation Steps

### Phase 1: Foundation & Layout
- **Setup Global Styles**: Define base styles for dark mode and typography in `index.css`.
- **Build Layout Components**:
  - **Header**: Sticky, Logo, Nav Links, Deposit Button.
  - **Sidebar**: Collapsible, Navigation Icons.
  - **Main Layout**: Responsive grid/flex structure to hold the Sidebar and Page Content.

### Phase 2: Core UI Components
- **Primitive Components**: Create reusable `Button` (variants: primary, outline, danger), `Input`, `Card`.
- **Feedback Components**: Implement `Skeleton` for loading states and `Toast` for notifications.

### Phase 3: Swap Feature Implementation
- **Data & Types**: Define Token interfaces and mock token list (BTC, ETH, USDT, etc.).
- **`useSwap` Hook**:
  - Implement mock exchange rate logic.
  - Handle `amount` changes and auto-calculate `to` amount.
  - Manage loading states and slippage.
- **Swap Card UI**:
  - **Tabs**: Market/Limit toggle.
  - **Inputs**: "From" and "To" fields with Token Selector.
  - **Token Modal**: Searchable list of tokens with "Hot Tokens".
  - **Details**: Accordion for Fees, Price Impact, Minimum Received.

### Phase 4: Integration & Refinement
- **Validation**: Integrate Zod to validate balance sufficiency and positive amounts.
- **Modals**: Implement "Confirm Swap" modal with summary.
- **Responsiveness**: Ensure mobile-first adaptation (collapsible sidebar, stacked layout).
- **Visual Polish**: Apply final "Binance" styling tweaks (hover states, transitions).

## 4. Verification
- **Build Check**: Run `npm run build` to ensure no TS errors.
- **Functional Test**:
  - Verify swap calculations.
  - Check validation errors (insufficient balance).
  - Confirm responsive behavior on different screen sizes.
